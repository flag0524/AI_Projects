## 1. 개요 (Overview)

마네킹(또는 모델) 이미지와 의류 이미지를 업로드하면, AI가 의류를 마네킹에 자연스럽게 입혀 합성 이미지를 생성하는 가상 피팅 시스템이다. 단순 합성이 아니라 **업로드한 이미지의 실제 사이즈 정보와 마네킹의 체형(비율·자세)을 함께 고려**하여, 옷이 늘어지거나 끼는 정도, 길이감, 드레이프(주름)까지 사실적으로 표현하는 것을 목표로 한다.

핵심 기술 방향은 **Diffusion 기반 가상 피팅(Virtual Try-On)** 으로, IDM-VTON·TryOnDiffusion 계열의 최신 생성 모델을 활용한다. 저사양 환경(CPU, VRAM 2GB 이하)에서는 경량 CV 워핑 엔진으로 자동 전환된다. 상세 기술 설계는 `TECHNICAL_PRD.md`를 참고한다.

## 2. 배경 및 문제 정의 (Background & Problem)

온라인 패션 커머스와 의류 제작 현장에서는 옷 한 벌을 보여주기 위해 실제 마네킹/모델 촬영이 필요하다. 이는 비용·시간이 크고, 사이즈별·체형별로 다시 촬영하기 어렵다. 또한 단순 누끼 합성은 옷의 길이·핏이 체형과 맞지 않아 비현실적으로 보인다.

해결하려는 문제:

- 촬영 없이 의류 이미지를 마네킹에 입혀 빠르게 룩북/상세컷을 생성하고 싶다.
- 옷의 **실제 치수**와 마네킹의 **체형/비율**을 반영해 핏(여유분·길이감)을 사실적으로 보여주고 싶다.
- 일반 의상·원피스·악세서리 카테고리별로 일관된 품질의 결과를 얻고 싶다.
- **GPU가 없거나 저사양 환경에서도** 기본적인 피팅 결과를 확인하고 싶다.

## 3. 목표 및 비목표 (Goals / Non-Goals)

### 3.1 목표

- 마네킹 이미지 + 의류 이미지 업로드 → 가상 피팅 결과 이미지 생성.
- 의류 이미지의 사이즈 메타데이터(실측 cm)와 마네킹 체형을 반영한 핏 표현.
- **3개 의류 대분류 지원**: 일반의상 / 원피스 / 기타 악세서리.
- 상의+하의 조합 레이어링 지원.
- 결과의 사실성(드레이프, 그림자, 원단 질감)과 원본 의류 디테일(로고·패턴) 보존.
- **GPU VRAM 2GB 이하 및 CPU 환경**에서도 CV 워핑 엔진으로 기본 피팅 가능.

### 3.2 비목표 (이번 범위 제외)

- 동영상·실시간 카메라 피팅.
- 실제 인체(고객 본인 사진) 대상 피팅.
- 의류 디자인·패턴 생성(생성형 디자인 도구) 기능.
- 결제·커머스 백오피스 연동.

## 4. 의류 카테고리 정의 (v1.3 개정)

| 프론트 표기 | 업로드 존 | 내부 코드 | 피팅 처리 방식 |
|---|---|---|---|
| 일반의상 — 상의 | 상의 전용 업로드 존 | `top` | 포즈·체형 정합 + Diffusion/CV 합성 |
| 일반의상 — 하의 | 하의 전용 업로드 존 | `bottom` | 포즈·체형 정합 + Diffusion/CV 합성 |
| 일반의상 — 상하의 동시 | 상의 + 하의 모두 업로드 | `top` + `bottom` (레이어링) | 상의 합성 후 하의 순차 합성 |
| 원피스 | 단일 업로드 존 | `dress` | 전신 정합 + Diffusion/CV 합성 |
| 기타 악세서리 | 단일 업로드 존 | `accessory` | 키포인트 기반 위치 결정 + 오버레이 합성 |

**일반의상 업로드 규칙:**
- 상의만 업로드 → 상의 단일 피팅
- 하의만 업로드 → 하의 단일 피팅
- 상의 + 하의 모두 업로드 → 자동 레이어링 전환 (별도 탭 이동 불필요)

> 악세서리는 체형-치수 정합(ease 계산)을 생략하고, 마네킹 키포인트 기반 위치에 자동 배치한다.

## 5. 대상 사용자 & 사용 시나리오 (Personas & Use Cases)

| 페르소나 | 니즈 | 핵심 시나리오 |
|---|---|---|
| 온라인 셀러 / MD | 빠른 상세컷 제작 | 신상 의류 이미지를 마네킹에 입혀 상품 페이지용 이미지 대량 생성 |
| 패션 브랜드 디자이너 | 핏/길이 사전 검토 | 시제품 의류를 표준 체형 마네킹에 올려 핏 확인 |
| 쇼핑몰 운영자 | 사이즈별 비교 | 동일 의류를 다른 체형 마네킹에 입혀 사이즈 가이드 제작 |
| 저사양 PC 사용자 | GPU 없이 미리보기 | CPU/저VRAM 환경에서 CV 워핑 엔진으로 빠른 피팅 확인 |

대표 플로우: 마네킹 업로드 → 카테고리 선택(라디오) → 의류 업로드(+사이즈 입력) → 핏 옵션 선택 → 생성 → 결과 확인/재생성 → 다운로드.

### 6.2 비기능 요구사항 (NFR)

- **성능**: GPU 기준 ≤ 15초(P95) / CPU Tier 3 ≤ 5초(P95).
- **품질**: Tier 1 사용자 평가 "사실적" ≥ 80% / Tier 3 ≥ 50%.
- **호환성**: GPU VRAM 2GB 이하 및 CPU-only 환경에서 Tier 3으로 동작.
- **확장성**: 비동기 작업 큐(Celery + Redis)로 수평 확장.
- **가용성**: 99.5% 이상.

## 8. UX / 화면 흐름 (요약)

1. **마네킹 업로드**: 드래그&드롭 또는 클릭.
2. **의류 종류 선택**: 라디오 버튼 (일반의상 / 원피스 / 기타 악세서리).
3. **의류 이미지 업로드**:
   - 일반의상: 상의 업로드 존 + 하의 업로드 존 분리 표시. 각각 독립적으로 업로드.
   - 원피스 / 악세서리: 단일 업로드 존.
4. **자동 레이어링 전환**: 일반의상에서 상의·하의 모두 업로드 시 안내 문구 표시 후 자동으로 레이어링 파이프라인 사용.
5. **치수 입력 패널**: 파일 업로드 후 해당 슬롯 아래 즉시 표시 (상의/하의 각각).
6. **옵션**: 핏 강도·후보 수·배경 제거.
7. **결과 화면**: 후보 그리드, 확대 보기, 재생성, 다운로드, 핏 분석.

## 11. 리스크 & 대응 (Risks)

- 의류 디테일 손실 → CLIP/DINOv2 자동 검증 + 재생성.
- 극단 체형/치수 불일치 → 여유율 경고 및 가드레일.
- 악세서리 위치 오배치 → 키포인트 신뢰도 임계 기반 fallback.
- Tier 3 품질 한계 → 결과 화면에 "CV 워핑 모드" 라벨 표시 및 GPU 업그레이드 안내.
- **저VRAM GPU(2GB 등)에서 디퓨전 구동 불가** → Tier 3 자동/수동 고정. 사진급 결과(Tier 1/2)는 8GB+ GPU(권장 12GB+) 또는 클라우드 GPU 필요.
- **드레스폼(다리·머리 없는 토르소) 마네킹** → 하의 배치 부정확(입힐 신체 부재). 상의/원피스 권장, 전신 마네킹 권장.
- **실사 의류 디테일(주름·소매·로고)** → Tier 3 CV 워핑은 평면적으로 단순화됨. 사실적 드레이프는 Tier 1 필요.
- 라이선스/저작권 → 사용자 권리 동의 및 정책 명시.
